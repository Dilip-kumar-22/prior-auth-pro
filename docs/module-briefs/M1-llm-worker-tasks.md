## Files to create

### 1. `backend/worker/schemas.py`

Pydantic v2 BaseModel classes used by the tasks for structured LLM output:

```python
from typing import Literal
from pydantic import BaseModel, Field


class ClinicalContext(BaseModel):
    """Distilled clinical picture extracted from a FHIR Bundle."""
    patient_age: int | None = None
    patient_sex: str | None = None
    primary_diagnosis_icd10: list[str] = Field(default_factory=list)
    secondary_diagnoses_icd10: list[str] = Field(default_factory=list)
    relevant_medications: list[str] = Field(default_factory=list)
    failed_prior_therapies: list[str] = Field(default_factory=list)
    lab_results_summary: str | None = None
    clinical_narrative: str = ""


class AuthDecision(BaseModel):
    """Gemini's prior-auth determination."""
    decision: Literal["approve", "deny", "pend"]
    reasoning: str = Field(..., description="3-5 sentences explaining the determination")
    confidence: float = Field(..., ge=0.0, le=1.0)
    key_factors: list[str] = Field(default_factory=list, description="Bullet-point factors weighed")
    cited_guidelines: list[str] = Field(default_factory=list, description="Policy codes referenced")
    required_documentation_missing: list[str] = Field(default_factory=list)


class AppealContext(BaseModel):
    """Inputs to the appeal letter generator."""
    denial_reason: str
    clinical_summary: str
    policy_citations: list[str]
    patient_age: int | None = None
    primary_diagnosis_icd10: str | None = None


class AppealLetter(BaseModel):
    """Structured output for appeal letter generation."""
    introduction: str
    clinical_justification: str = Field(..., description="Paragraph tying patient facts to coverage criteria")
    policy_citations: str = Field(..., description="Paragraph citing relevant guidelines")
    conclusion: str
```

### 2. `backend/worker/llm_client.py`

Wrapper around Gemini with structured output support. ~250-300 LOC.

**Class `GeminiClient`:**

- `__init__(api_key: str, model: str = "gemini-3.1-pro")`
- `async generate_structured(prompt: str, schema: type[BaseModel], max_tokens: int = 4096, temperature: float = 0.2) -> BaseModel`
  - Use `google-genai` SDK's `response_schema` param (or JSON mode + pydantic parse as fallback).
  - Retry on 503/429 with exponential backoff (3 tries: 2s, 4s, 8s).
  - On persistent 503: fall back through chain: `gemini-3.1-pro` → `gemini-2.5-pro` → `gemini-3-flash` → `gemini-2.5-flash`. Log every fallback with `logger.warning`.
- `async generate_text(prompt: str, max_tokens: int = 2048, temperature: float = 0.3) -> str`
  - Unstructured text generation with same retry/fallback behavior.

**Module-level helpers** (these are what `tasks.py` calls):

- `async generate_auth_decision(clinical: ClinicalContext, relevant_guidelines: list[dict]) -> AuthDecision` — builds the prompt, calls `GeminiClient.generate_structured(schema=AuthDecision)`, returns decision.
- `async generate_appeal_letter(ctx: AppealContext) -> AppealLetter` — same pattern, schema=AppealLetter.

**Prompts:** Keep the prompt strings as module-level constants for now (`AUTH_DECISION_PROMPT_TEMPLATE`, `APPEAL_LETTER_PROMPT_TEMPLATE`). Module 2 will externalize these to jinja2 templates — don't pre-optimize.

**Observability:** Every call logs: model used, prompt length, response length, latency_ms, fallback path (if any). Use `logger = logging.getLogger(__name__)`.

### 3. `backend/worker/tasks.py`

Two ARQ background tasks. ~300-400 LOC total. Must match the signatures already registered in `worker/main.py::WorkerSettings.functions`.

**Task: `process_auth_request_task(ctx, auth_request_id: int) -> dict`**

Flow:
1. Load `AuthRequest` by ID from DB (use `ctx["db_engine"]` which is a SQLAlchemy async engine — see `worker/main.py` startup).
2. Append `AuthEvent(event_type="REQUEST_RECEIVED", auth_request_id=...)`.
3. **FHIR extract:**
   - `from fhir.client import client_from_session` — use session-bound client.
   - Fetch FHIR Bundle for `auth_request.patient_fhir_id`.
   - Pass to `fhir.resources.extract_clinical_context(bundle) -> ClinicalContext`.
4. Append `AuthEvent(event_type="CLASSIFIED", metadata={"service_category": ...})`.
5. **Rules engine:**
   - `from engines.rules.engine import RulesEngine`
   - `engine = RulesEngine(session)` (async session)
   - `rules_result = await engine.evaluate(auth_request, clinical)` — returns one of: `auto_approve`, `auto_deny`, `ai_review`
6. Append `AuthEvent(event_type="RULES_EVALUATED", metadata=rules_result)`.
7. If `rules_result.verdict == "auto_approve"`: set `auth_request.status = "approved"`, create `Decision`, commit, return early.
8. If `rules_result.verdict == "auto_deny"`: set status = "denied", create `Decision`, commit, return early.
9. If `rules_result.verdict == "ai_review"`:
   - **RAG:** `from engines.rag.engine import RagEngine`; `guidelines = await rag.search(clinical.clinical_narrative, top_k=5)`.
   - Append `AuthEvent(event_type="RAG_SEARCHED", metadata={"top_k": 5, "scores": [...]})`.
   - **Gemini decide:** `decision = await generate_auth_decision(clinical, guidelines)`.
   - Append `AuthEvent(event_type="AI_DECIDED", metadata=decision.model_dump())`.
   - Persist `Decision(auth_request_id=..., verdict=decision.decision, reasoning=..., confidence=..., cited_guideline_ids=...)`.
   - Set `auth_request.status` based on decision.
10. Append `AuthEvent(event_type="DECISION_FINALIZED", metadata={...})`.
11. Emit WebSocket event via `api.websocket.emit_auth_update(auth_request_id, event)` (the function exists in existing code — use it).
12. Return `{"status": auth_request.status, "decision_id": decision.id, "latency_ms": int((time.time() - start) * 1000)}`.

**Idempotency:** At task start, query `AuthEvent` for `event_type == "DECISION_FINALIZED" AND auth_request_id == <id>`. If found, return early with cached result — task was already processed and ARQ redelivered.

**Error handling:** Wrap the whole flow in try/except. On exception: append `AuthEvent(event_type="ERROR", metadata={"exception": repr(e), "traceback": traceback.format_exc()})`, set `auth_request.status = "error"`, commit, re-raise so ARQ records the failure.

**Task: `generate_appeal_task(ctx, appeal_id: int) -> dict`**

Flow:
1. Load `Appeal` + linked denied `AuthRequest` + associated `Decision`.
2. Build `AppealContext` with denial_reason, clinical_summary (reconstruct from stored `AuthEvent`s or from persisted `ClinicalContext`), policy_citations, patient_age, primary_diagnosis.
3. `letter = await generate_appeal_letter(ctx)` from `llm_client.py`.
4. Render to Markdown: concatenate `letter.introduction + "\n\n" + letter.clinical_justification + "\n\n" + letter.policy_citations + "\n\n" + letter.conclusion`.
5. Persist `appeal.letter_markdown = <rendered>`.
6. Set `appeal.status = "generated"`, `appeal.generated_at = now()`.
7. Commit.
8. Return `{"status": "generated", "appeal_id": appeal_id, "letter_length": len(rendered)}`.

Also idempotent: if `appeal.letter_markdown` is already non-empty and `status == "generated"`, return cached result.
